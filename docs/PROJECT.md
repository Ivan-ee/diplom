# Project Overview

Bakery — интернет-магазин кондитерской с упором на интерактивный 3D-конструктор тортов. Пользователь может выбрать готовые товары из каталога или собрать индивидуальную конфигурацию торта, добавить ее в корзину и оформить заявку на самовывоз.

Документ описывает текущее устройство проекта для разработчиков. Доменные термины зафиксированы отдельно в [../CONTEXT.md](../CONTEXT.md).

## Стек

- Monorepo: pnpm workspaces, Turborepo.
- Web: Next.js 16, React 19, Tailwind CSS, Zustand, React Hook Form, Zod.
- 3D: Three.js, React Three Fiber, Drei, GLB-модели.
- API: NestJS 11, Swagger, class-validator, cookie-based JWT auth, CSRF middleware, throttling.
- Database: PostgreSQL, Drizzle ORM, Drizzle migrations.
- Storage: MinIO для product images, textures, screenshots и models.
- Search: Meilisearch для полнотекстового поиска по товарам.
- Tests: Vitest для web, API и отдельных library/store сценариев.

## Структура

- `apps/web` — пользовательский интерфейс магазина, конструктора, checkout, личного кабинета и админки.
- `apps/api` — REST API для каталога, конструктора, заказов, загрузок, промокодов, поиска и админских операций.
- `packages/db` — схема БД, миграции, seed-данные, загрузка ассетов в MinIO.
- `packages/shared-types` — общие TypeScript-типы для API и web.
- `apps/web/public/models` и `Cakes` — GLB-модели для визуализации конструктора.

## Пользовательские сценарии

- Просмотр главной страницы, каталога и карточек товаров.
- Добавление готового товара в корзину.
- Сборка торта в 3D-конструкторе: форма, количество ярусов, основа, начинка, покрытие, декор, свеча и надпись.
- Добавление конфигурации торта в корзину как отдельной позиции заказа.
- Оформление заявки на самовывоз с датой, временным слотом, телефоном, комментарием и промокодом.
- Просмотр истории заказов в личном кабинете.
- Администрирование товаров, заказов, пользователей, промокодов и данных конструктора.

## Архитектура

Web-приложение работает через Next.js. Клиентские запросы идут на `/api/*` и проксируются rewrites в NestJS API. Серверные запросы используют `NEXT_PUBLIC_API_URL`.

API оборачивает ответы через общий response interceptor и валидирует входные DTO через глобальный `ValidationPipe`. Ошибки приводятся к единому формату фильтрами исключений. Авторизация основана на JWT в httpOnly cookie `bakery_token`; защищенные state-changing запросы дополнительно используют CSRF-токен.

PostgreSQL хранит пользователей, товары, категории, заказы, позиции заказов, данные конструктора, промокоды и отзывы. Drizzle-схемы находятся в `packages/db/src/schema`, миграции — в `packages/db/drizzle`.

MinIO хранит публично читаемые изображения и ассеты. API выдает presigned PUT URL через `POST /api/upload/presign`; web загружает файл напрямую в MinIO и сохраняет публичный URL в бизнес-данных, например screenshot конструктора.

Meilisearch индексирует товары при старте search module и используется для поиска по названию и описанию.

## 3D-конструктор

Публичный маршрут конструктора — `/constructor`; Next.js rewrite ведет его на страницу `/cake-constructor`.

Основной state конструктора хранится в Zustand store `useConstructorStore`:

- `shape` — форма торта: круг, квадрат, сердце.
- `tierCount` и `layers` — количество ярусов, основы, начинки и вес каждого яруса.
- `coating` — тип и вариант покрытия, режим цвета, потеки.
- `activeDecorations` и `selectedDecorations` — визуальные варианты декора и связанные ingredient IDs для расчета.
- `hasCandle` и `inscription` — свеча и надпись.
- `totalPrice` — клиентская оценка цены для интерфейса.

Данные конструктора приходят из `GET /api/constructor/ingredients`. Если API недоступен, web использует mock-данные из `apps/web/src/lib/constructor/mock-ingredients.ts`, чтобы интерфейс оставался работоспособным для разработки.

Серверный расчет цены выполняется через `POST /api/constructor/calculate` и повторно вызывается при создании заказа. Клиентская цена не считается доверенной.

При добавлении торта в корзину web пытается сделать screenshot текущего WebGL canvas и загрузить его в MinIO bucket `screenshots`. Если upload не удался, позиция все равно может быть добавлена с fallback image.

## Заказы и checkout

Корзина хранится на клиенте в Zustand store `useCartStore`. В корзине есть два типа позиции:

- `product` — готовый товар каталога с `productId`.
- `constructor` — индивидуальная конфигурация торта с `cakeConfig`.

Checkout создает заказ через `POST /api/orders`. API пересчитывает цену каждой позиции на сервере:

- для `product` цена берется из актуального товара;
- для `constructor` цена считается через `ConstructorService.calculatePrice`;
- промокод валидируется на сервере перед записью заказа.

Текущий checkout — это заявка на самовывоз. В системе есть дата получения, временной слот, телефон и комментарий. Онлайн-оплата и отдельная модель доставки сейчас не реализованы.

## Админка

Админские страницы находятся под `/admin/*` и защищены проверкой роли `admin`. Middleware на web стороне делает UX-защиту маршрутов, а фактическая проверка прав выполняется в NestJS через `JwtAuthGuard` и `RolesGuard`.

В админке доступны:

- управление заказами и статусами;
- управление товарами и фотографиями;
- управление пользователями;
- управление промокодами;
- управление данными конструктора: основы, начинки, покрытия, декор.

## Локальная разработка

Базовый Docker-поток:

```bash
cp .env.example .env
docker compose up -d postgres minio minio-init meilisearch
docker compose --profile seed run --rm --build seed
docker compose up -d --build
```

Dev-поток через Makefile:

```bash
make install
make dev
```

Проверки:

```bash
make test
make type-check
pnpm build
```

Для docs-only изменений достаточно проверить Markdown глазами и выполнить `git diff --check`.

## Текущие ограничения

- Онлайн-оплата не реализована.
- Доставка не выделена в отдельную модель; checkout оформляет заявку на самовывоз.
- Конструктор хранит часть визуальных параметров на клиенте, а сервер пересчитывает только бизнес-значимые ингредиенты и цену.
- Screenshot конструктора является вспомогательным изображением заказа; сбой загрузки screenshot не блокирует добавление конфигурации в корзину.
- Данные конструктора редактируются администратором, но GLB-модели и визуальный registry требуют аккуратного соответствия между модельными вариантами и бизнес-ингредиентами.
