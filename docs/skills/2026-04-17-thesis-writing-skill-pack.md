# Thesis Writing Skill Pack

Дата: 2026-04-17

## Что установлено в `~/.codex/skills`

### 1. `writing-anti-ai`
- Источник: `Galaxy-Dawn/claude-scholar`, ветка `codex`
- Путь: `skills/writing-anti-ai`
- Задача: убирать AI-паттерны, роботизированные конструкции, лишние связки, filler phrases и шаблонные риторические ходы.
- Зачем для диплома: это основной skill для финальной полировки главы, чтобы текст звучал естественнее и менее шаблонно.

### 2. `doc-coauthoring`
- Источник: `Galaxy-Dawn/claude-scholar`, ветка `codex`
- Путь: `skills/doc-coauthoring`
- Задача: пошагово собирать большой документ через структуру, уточнения, секции и итеративную редактуру.
- Зачем для диплома: подходит для написания главы как длинного цельного документа, а не набора случайных абзацев.

### 3. `paper-self-review`
- Источник: `Galaxy-Dawn/claude-scholar`, ветка `codex`
- Путь: `skills/paper-self-review`
- Задача: проверять полноту, логику, структуру, ясность, качество аргументации и общую академическую связность.
- Зачем для диплома: useful как финальный контроль качества перед сдачей или перед отправкой на проверку научруку.

### 4. `citation-verification`
- Источник: `Galaxy-Dawn/claude-scholar`, ветка `codex`
- Путь: `skills/citation-verification`
- Задача: проверять корректность ссылок, библиографии и не допускать фейковых или кривых цитат.
- Зачем для диплома: критично для академического текста, чтобы не было выдуманных источников и ошибок в ссылках.

## Что нашёл, но не ставил

### `humanizer`
- Источник: `https://github.com/blader/humanizer`
- Причина интереса: очень популярный skill для анти-AI-редактуры; на 2026-04-17 у репозитория было `14213` stars.
- Почему не ставил автоматически:
  - skill размечен скорее под Claude Code/OpenCode, а не явно под Codex;
  - он во многом пересекается с уже установленным `writing-anti-ai`;
  - для твоего сценария важнее иметь связку “структура + ссылки + self-review + anti-AI polish”, чем два почти одинаковых humanizer-skill.

### `scientific-writing` / `claude-scientific-writer`
- Источник: `https://github.com/K-Dense-AI/claude-scientific-writer`
- Причина интереса: сильный академический writing toolkit; на 2026-04-17 у репозитория было `1536` stars.
- Почему не ставил:
  - набор сильно заточен под paper workflow, journal submission и research tooling;
  - в skill’ах много слишком жёстких допущений для обычной главы диплома;
  - есть навязанные шаги вроде обязательных figure/schematic workflow, которые для твоей задачи сейчас скорее мешают.

### `ux-writing-skill`
- Источник: `https://github.com/content-designer/ux-writing-skill`
- Причина интереса: системный и качественный editing framework, на 2026-04-17 у репозитория было `77` stars.
- Почему не ставил:
  - это skill для интерфейсного текста и microcopy;
  - для дипломной главы почти не даёт пользы по сравнению с академическими skills.

## Рекомендуемый порядок использования для 2 главы

1. Сначала `doc-coauthoring`
- Собрать структуру главы.
- Зафиксировать секции, тезисы, переходы, логику аргументации.

2. Потом обычная черновая генерация текста
- Писать абзацы по твоим материалам, а не “с нуля из воздуха”.
- Держать опору на реальные данные, стек, решения, архитектуру и источники.

3. Потом `citation-verification`
- Проверять все ссылки, даты, авторов, названия работ и корректность упоминаний.

4. Потом `paper-self-review`
- Прогонять главу на полноту, связность, академический стиль, логику и качество доказательности.

5. Только в конце `writing-anti-ai`
- Убирать шаблонность.
- Делать текст живее, плотнее и естественнее.
- Не использовать это как замену смысла и структуры.

## Важное замечание

Этот набор нужен для:
- улучшения качества текста;
- уменьшения AI-slop;
- повышения естественности формулировок;
- контроля структуры и ссылок.

Этот набор **не должен использоваться как инструмент для обмана проверок**. Лучший “анти-AI” результат получается не от маскировки, а от нормального процесса:
- сначала твоя логика;
- потом твои данные;
- потом человеческая редактура;
- и только затем анти-шаблонная полировка.
